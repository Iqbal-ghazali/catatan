1. Resize Volume dari Dalam VM

Setelah memperluas disk di sisi hypervisor atau cloud provider, gunakan growpart untuk memperluas partisi tanpa menghapus data. Pastikan paket cloud-guest-utils sudah terinstall: sudo apt install cloud-guest-utils -y

sudo growpart {partition_device} {nomor_partisi}

# Contoh: perluas partisi ke-1 pada /dev/xvda
sudo growpart /dev/xvda 1

Verify:

lsblk

2. Resize Filesystem — Keluarga EXT

Setelah partisi diperluas, filesystem harus ikut di-resize. Untuk ext2 / ext3 / ext4 gunakan resize2fs.

sudo resize2fs {partition}

# Contoh
sudo resize2fs /dev/xvda1
Tips: Jika muncul error "bad magic number", jalankan e2fsck -f {partition} terlebih dahulu untuk memperbaiki filesystem.

3. Resize Filesystem — XFS

Untuk filesystem XFS, gunakan xfs_growfs. Target berupa mount point, bukan device path.

sudo xfs_growfs {target/mount_point}

# Contoh: filesystem di-mount di /
sudo xfs_growfs /

# Contoh: filesystem di-mount di /data
sudo xfs_growfs /data

Verify (berlaku untuk ext maupun xfs):

df -h

4. Format Partisi Baru dengan mkfs

Saat menambahkan disk baru, format terlebih dahulu sebelum bisa di-mount.

sudo mkfs -t {type} {partition}

# Contoh untuk ext4
sudo mkfs -t ext4 /dev/xvdd1

# Contoh untuk xfs
sudo mkfs -t xfs /dev/xvdd1

Verify filesystem & cek UUID:

blkid {partition}

# Contoh output:
# /dev/xvdd1: UUID="a1b2c3d4-..." TYPE="ext4"

5. Buat Mount Point & Mount Filesystem

sudo mkdir /data
sudo mount /dev/xvdd1 /data

Verify:

lsblk
df -h

6. Persistent Mount via /etc/fstab

Tambahkan entri ke /etc/fstab menggunakan UUID agar filesystem otomatis ter-mount saat boot.

# Format
UUID="<uuid-device>"  {mount_point}  {type}  defaults  0 0

# Contoh
UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890"  /data  ext4  defaults  0 0
Warning: Kesalahan di /etc/fstab bisa mencegah sistem boot. Selalu test dengan sudo mount -a sebelum reboot. 
# Test entri fstab tanpa reboot
sudo mount -a

# Verify
df -h

1. Setup VM server-mon sebagai Local Repository Server (Case: NGINX)

a. Install dan konfigurasi nginx sebagai HTTP repo server

apt update
apt install -y nginx
systemctl enable nginx
systemctl start nginx

b. Buat direktori /mirror/ubuntu-additional

mkdir -p /mirror/ubuntu-additional/master-sync
mkdir -p /mirror/ubuntu-additional/gpg

c. Download GPG key nginx

wget https://nginx.org/keys/nginx_signing.key \
-O /mirror/ubuntu-additional/gpg/nginx_signing.key

d. Sync repo nginx menggunakan apt-mirror

Semua base repo harus dinonaktifkan. Isi file /etc/apt/mirror.list hanya sebagai berikut:

set base_path    /mirror/ubuntu-additional/master-sync
set nthreads     20
set _tilde 0
deb https://nginx.org/packages/ubuntu noble nginx
clean https://nginx.org/packages/ubuntu

Jalankan proses mirror:

apt-mirror

e. Symlink /mirror/ubuntu-additional ke /var/www/html

ln -s /mirror/ubuntu-additional /var/www/html/ubuntu-additional

f. Symlink hasil sync nginx ke /mirror/ubuntu-additional/nginx

Struktur hasil mirror dari nginx berada di:

/mirror/ubuntu-additional/master-sync/mirror/nginx.org/packages/ubuntu/

Buat direktori nginx dan lakukan symlink:

mkdir -p /mirror/ubuntu-additional/nginx
ln -s /mirror/ubuntu-additional/master-sync/mirror/nginx.org/packages/ubuntu/dists \
/mirror/ubuntu-additional/nginx/dists
ln -s /mirror/ubuntu-additional/master-sync/mirror/nginx.org/packages/ubuntu/pool \
/mirror/ubuntu-additional/nginx/pool

2. Test Instalasi NGINX di VM client-mon

a. Import GPG key nginx di client-mon

wget http://10.10.10.11/ubuntu-additional/gpg/nginx_signing.key -O nginx.key
gpg --dearmor nginx.key > /usr/share/keyrings/nginx.gpg

b. Membuat file repo menggunakan EOF

cat << 'EOF' > /etc/apt/sources.list.d/nginx.list
deb [trusted=yes arch=amd64] http://10.10.10.11/ubuntu-additional/nginx noble nginx
EOF

c. Install nginx dari local repository

apt update
apt install -y nginx

2. Pre-Check Sebelum Reboot

Sebelum melakukan reboot, jalankan script berikut di dalam VM untuk mendokumentasikan kondisi awal — hostname, uptime, network, disk, dan fstab.

bash <<'EOF'
echo "===== 1.1 Hostname & Uptime ====="
echo "Hostname : $(hostname)"
echo "Uptime   : $(uptime -p)"
echo -e "===== 1.2 Network ====="
echo "IP Address:"
ip -4 -o addr show | awk '{print $2 ": " $4}'
echo "Route:"
ip route show 
echo -e "===== 1.3 Disk ====="
echo -e "===== df -h ====="
df -h --output=source,size,used,avail,pcent,target
echo -e "===== lsblk ====="
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT 
echo -e "===== cat /etc/fstab ====="
grep -vE '^#|^$' /etc/fstab 
EOF

3. Reboot Instance

a. Reboot dari dalam VM

reboot

b. Reboot via CLI OpenStack (dari controller)

Source credential terlebih dahulu, lalu lakukan soft reboot. Gunakan hard reboot jika soft reboot tidak berhasil.

source admin.sh

1. Persiapan Awal

Update dan upgrade sistem terlebih dahulu sebelum memulai instalasi.

apt-get update -y; apt-get upgrade -y

2. Install Node Exporter

a. Buat user node_exporter

sudo useradd --no-create-home --shell /bin/false node_exporter

b. Download Node Exporter ke /mnt

wget https://github.com/prometheus/node_exporter/releases/download/v1.6.1/node_exporter-1.6.1.linux-amd64.tar.gz -P /mnt

c. Masuk ke /mnt dan ekstrak file

cd /mnt
tar -zxvf node_exporter-1.6.1.linux-amd64.tar.gz

d. Copy binary ke /usr/local/bin

cp /mnt/node_exporter-1.6.1.linux-amd64/node_exporter /usr/local/bin

e. Set ownership ke user node_exporter

chown node_exporter:node_exporter /usr/local/bin/node_exporter

f. Buat systemd service

Buat file service di /etc/systemd/system/node_exporter.service:

nano /etc/systemd/system/node_exporter.service
[Unit]
Description=Node Exporter
Wants=network-online.target
After=network-online.target

[Service]
User=node_exporter
Group=node_exporter
Type=simple
ExecStart=/usr/local/bin/node_exporter

[Install]
WantedBy=multi-user.target

g. Enable dan jalankan service

systemctl daemon-reload; systemctl start node_exporter; systemctl enable node_exporter; systemctl status node_exporter

h. Verify Node Exporter

curl {ip}:9100

3. Install Prometheus

a. Buat user prometheus

sudo useradd --no-create-home --shell /usr/sbin/nologin prometheus

b. Download Prometheus ke /mnt

wget https://github.com/prometheus/prometheus/releases/download/v2.46.0/prometheus-2.46.0.linux-amd64.tar.gz -P /mnt

c. Masuk ke /mnt dan ekstrak

cd /mnt
tar -zxvf prometheus-2.46.0.linux-amd64.tar.gz

d. Copy binary prometheus dan promtool ke /usr/local/bin

cp /mnt/prometheus-2.46.0.linux-amd64/prometheus /usr/local/bin/
cp /mnt/prometheus-2.46.0.linux-amd64/promtool /usr/local/bin/

e. Set ownership binary

chown prometheus:prometheus /usr/local/bin/prom*

f. Buat direktori /etc/prometheus dan copy file konfigurasi

mkdir /etc/prometheus
cp /mnt/prometheus-2.46.0.linux-amd64/console* /etc/prometheus/ -Rf
cp /mnt/prometheus-2.46.0.linux-amd64/prometheus.yml /etc/prometheus
chown prometheus:prometheus /etc/prometheus/ -Rf

4. Konfigurasi Prometheus

Edit file /etc/prometheus/prometheus.yml dan tambahkan job untuk node_exporter. Sesuaikan IP target dengan IP VM yang menjalankan node_exporter.

scrape_configs:
  # Job default — Prometheus scrape dirinya sendiri
  - job_name: "prometheus"
    static_configs:
      - targets: ["localhost:9090"]

  # Job baru — scrape node_exporter di VM target
  - job_name: "node_exporter"
    static_configs:
      - targets: ["192.168.1.7:9100"]

5. Jalankan Prometheus sebagai Service

a. Buat systemd service

Buat file /etc/systemd/system/prometheus.service:

nano /etc/systemd/system/prometheus.service
[Unit]
Description=Prometheus Monitoring
Wants=network-online.target
After=network-online.target

[Service]
User=prometheus
Group=prometheus
Type=simple
ExecStart=/usr/local/bin/prometheus \
  --config.file /etc/prometheus/prometheus.yml \
  --storage.tsdb.path /etc/prometheus/data/ \
  --web.console.templates=/etc/prometheus/consoles \
  --web.console.libraries=/etc/prometheus/console_libraries
ExecReload=/bin/kill -HUP $MAINPID

[Install]
WantedBy=multi-user.target

b. Enable dan jalankan service

systemctl daemon-reload; systemctl start Prometheus; systemctl enable Prometheus; systemctl status prometheus
Akses Prometheus UI: Setelah service berjalan, buka browser dan akses http://{ip-prometheus}:9090 untuk masuk ke dashboard Prometheus. Cek menu Status → Targets untuk memastikan node_exporter terdaftar dan statusnya UP.

1. Tentang VictoriaMetrics

VictoriaMetrics adalah time-series database (TSDB) berkinerja tinggi yang kompatibel dengan Prometheus. Bisa digunakan sebagai pengganti storage bawaan Prometheus dengan konsumsi RAM dan disk yang lebih efisien, terutama untuk data skala besar.

Kenapa VictoriaMetrics? Kompresi data lebih baik, query PromQL tetap kompatibel, dan mendukung remote write/read dari Prometheus tanpa perubahan konfigurasi besar.

2. Install VictoriaMetrics

a. Download binary

wget https://github.com/VictoriaMetrics/VictoriaMetrics/releases/latest/download/victoria-metrics-linux-amd64-prod.tar.gz
tar -xzf victoria-metrics-linux-amd64-prod.tar.gz
sudo mv victoria-metrics-prod /usr/local/bin/victoriametrics

b. Buat direktori data

sudo mkdir -p /var/lib/victoriametrics

3. Jalankan sebagai Service

a. Buat file systemd service

sudo vi /etc/systemd/system/victoriametrics.service

Isi file service dengan konfigurasi berikut:

[Unit]
Description=VictoriaMetrics
After=network.target

[Service]
User=root
ExecStart=/usr/local/bin/victoriametrics \
  --storageDataPath=/var/lib/victoriametrics \
  --httpListenAddr=:8428
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

b. Enable dan jalankan service

sudo systemctl daemon-reload
sudo systemctl enable victoriametrics
sudo systemctl start victoriametrics

c. Verifikasi service berjalan

sudo systemctl status victoriametrics
curl http://localhost:8428/health
Expected output: curl ke /health akan mengembalikan OK jika VictoriaMetrics berjalan normal.

4. Migrasi Data Lama ke VictoriaMetrics menggunakan vmctl

a. Buat snapshot data Prometheus yang lama

curl -XPOST http://localhost:9090/api/v1/admin/tsdb/snapshot

Expected output:

{"status":"success","data":{"name":"20240101T000000Z-abc123"}}
Catat nama snapshot yang muncul di output (contoh: 20240101T000000Z-abc123). Snapshot disimpan di direktori: /var/lib/prometheus/snapshots/<nama_snapshot>

b. Install vmctl

wget https://github.com/VictoriaMetrics/VictoriaMetrics/releases/latest/download/vmutils-linux-amd64.tar.gz
tar -xzf vmutils-linux-amd64.tar.gz
sudo mv vmctl /usr/local/bin/

c. Jalankan migrasi

Ganti {nama-snapshot} dengan nama snapshot yang sudah dicatat sebelumnya:

vmctl prometheus \
  --prom-snapshot=/var/lib/prometheus/snapshots/{nama-snapshot} \
  --vm-addr=http://localhost:8428 \
  --prom-concurrency=4

d. Verifikasi data berhasil dimigrasikan

curl "http://localhost:8428/api/v1/query?query=up"

Atau akses UI VictoriaMetrics di browser:

http://{ip-server}:8428/vmui
Pastikan metric-metric dari Prometheus lama sudah muncul di vmui. Coba query sederhana seperti up atau node_cpu_seconds_total untuk konfirmasi.

1. Persiapan

Pastikan Prometheus sudah terinstall di compute-2 dan diharapkan versinya sama dengan yang ada di compute-1. Cek versi di masing-masing node: 

prometheus --version
⚠️ Perhatian: Perbedaan versi Prometheus antar node bisa menyebabkan inkompatibilitas format data. Usahakan versi sama sebelum melanjutkan.

2. Copy File Konfigurasi dari compute-1 ke compute-2

a. Copy file prometheus.yml

scp /etc/prometheus/prometheus.yml user@compute-2:/etc/prometheus/

b. Copy direktori rules

scp -r /etc/prometheus/rules/ user@compute-2:/etc/prometheus/rules/

3. Migrasi Data Lama dari compute-1

a. Buat snapshot data Prometheus di compute-1

curl -XPOST http://localhost:9090/api/v1/admin/tsdb/snapshot
Catat nama snapshot dari output, contoh: 20240101T000000Z-abc123. Snapshot tersimpan di /var/lib/prometheus/snapshots/<nama_snapshot>

b. Copy snapshot ke compute-2 menggunakan rsync

rsync -avz --progress /var/lib/prometheus/snapshots/{nama-snapshot} root@15.15.15.103:/var/lib/prometheus/

c. Pindahkan data snapshot ke direktori utama Prometheus di compute-2

Login ke compute-2, lalu jalankan:

cd /var/lib/prometheus/{nama-snapshot}
mv * ..

d. Fix ownership direktori data

chown -R prometheus:prometheus /var/lib/prometheus/

4. Restart & Verifikasi Prometheus di compute-2

systemctl daemon-reload
systemctl start prometheus
systemctl enable prometheus
systemctl status prometheus
curl http://localhost:9090/-/healthy
Buka http://{ip-compute-2}:9090 dan cek menu Status → Targets untuk memastikan semua target scrape kembali UP.

5. Matikan Prometheus di compute-1

Setelah semua diverifikasi dan tidak ada masalah di compute-2:

systemctl stop prometheus
systemctl disable prometheus
⚠️ Pastikan Prometheus di compute-2 sudah berjalan stabil dan semua data terverifikasi sebelum mematikan service di compute-1.

a. Buat file systemd service

nano /etc/systemd/system/loki.service
[Unit]
Description=Loki service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/loki -config.file /etc/loki-config.yaml

[Install]
WantedBy=multi-user.target

b. Enable dan jalankan service

systemctl daemon-reload; systemctl enable --now loki.service

c. Verifikasi Loki berjalan

systemctl status loki.service
curl localhost:3100/ready
Expected output: Endpoint /ready akan merespons ready jika Loki sudah siap menerima log.

3. Install Promtail

a. Download dan extract binary Promtail

curl -LO "https://github.com/grafana/loki/releases/latest/download/promtail-linux-amd64.zip"
unzip promtail-linux-amd64.zip
sudo mv promtail-linux-amd64 /usr/local/bin/promtail
sudo chmod +x /usr/local/bin/promtail

b. Buat file konfigurasi Promtail

Buat file /etc/promtail-config.yaml. Ganti [ip-loki] dan [nama-host] sesuai environment:

server:
  http_listen_port: 9080
  grpc_listen_port: 0

positions:
  filename: /data/loki/positions.yaml

clients:
  - url: http://[ip-loki]:3100/loki/api/v1/push

scrape_configs:
  - job_name: system-logs
    static_configs:
      - targets:
          - localhost
        labels:
          job: varlogs-pod-prome
          host: [nama-host]
          __path__: /var/log/*.log
Catatan: __path__ menentukan file log yang akan di-scrape oleh Promtail. Sesuaikan path jika log aplikasi berada di lokasi lain, misalnya /var/log/nginx/*.log.

4. Jalankan Promtail sebagai Service

a. Buat file systemd service

nano /etc/systemd/system/promtail.service
[Unit]
Description=Promtail service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/promtail -config.file /etc/promtail-config.yaml

[Install]
WantedBy=multi-user.target

b. Enable dan jalankan service

systemctl daemon-reload; systemctl enable --now promtail.service

c. Verifikasi Promtail berjalan

systemctl status promtail.service
Setelah Promtail berjalan, log dari /var/log/*.log akan mulai dikirim ke Loki secara otomatis. Untuk memverifikasi, tambahkan Loki sebagai data source di Grafana dengan URL http://[ip-loki]:3100 lalu query menggunakan LogQL.
